# Dream Machine

## Project Overview

Dream Machine is an interactive AI artwork that explores how a machine might interpret and reconstruct human dreams.

Users enter a dream description which is analysed by the system using machine learning and custom text-processing techniques. The machine then performs a visible reconstruction process before generating an abstract visual interpretation.

The project investigates the relationship between human imagination and machine interpretation, questioning whether artificial intelligence can truly understand subjective experiences such as dreams.

---

## Concept

Dreams are emotional, symbolic and deeply personal experiences. Humans understand dreams through memory, feeling and subconscious associations, whereas machines process information through data and pattern recognition.

Dream Machine explores what happens when a machine attempts to reconstruct a dream despite having no genuine understanding of dreaming itself.

Instead of recreating dreams realistically, the artwork translates dream descriptions into computational forms and abstract visual structures. The resulting output is not intended to be the user's actual dream, but rather the machine's interpretation of it.

---

## Machine Learning

The project uses the ml5.js Sentiment Analysis model as its machine learning component.

When a dream is submitted, the text is analysed by the model while the system simultaneously identifies symbolic dream imagery through custom JavaScript logic. Keywords and recurring dream motifs such as oceans, moons, forests, staircases and movement influence the final reconstruction.

The project intentionally combines machine learning with rule-based interpretation to highlight how AI systems often simplify complex human experiences into patterns, classifications and recognised symbols.

---

## Artistic Approach

The artwork is designed to resemble a machine attempting to dream.

After the user submits a dream, the system enters a reconstruction phase where code-like visual activity suggests that the machine is analysing and processing the text. The dream is then transformed into an abstract visual composition generated from recognised symbols and computational rules.

The work focuses on interpretation rather than accuracy. Rather than reproducing the dream itself, the machine produces its own simplified version based on the information it can recognise and process.

---

## Critical Reflection

Dream Machine explores the limitations of artificial intelligence and challenges assumptions about machine creativity.

Although AI systems can generate convincing outputs, they do not possess memories, emotions or lived experiences. The machine can identify patterns and symbols within a dream description, but it cannot truly experience dreaming.

The project therefore positions AI as a system of interpretation rather than understanding. The visual reconstruction reveals how machines reduce subjective human experiences into computational structures and symbolic patterns.

## AI Disclosure

This project uses the pre-trained Sentiment Analysis model provided by ml5.js.

The machine learning model analyses user-submitted dream descriptions as part of the interpretation process. Visual outputs are generated through custom JavaScript systems that identify symbolic dream imagery and translate those patterns into abstract visual forms.

No generative image models, large language models, or external AI APIs are used within the final artwork.

The project critically examines the limitations of AI interpretation and does not present machine-generated outputs as objective representations of human dreams.
